================================================================================
                              PROJECT OVERVIEW
================================================================================

AI Doctor Pro is a comprehensive full-stack application that combines:
- Multi-Role Architecture (Patient and Doctor Workflows)
- Speech-to-Text (STT) for capturing patient symptoms via voice
- Medical Vision Analysis for analyzing patient-uploaded images
- Intelligent Triage System to route cases to appropriate specialists
- Diagnosis Engine with RAG (Retrieval-Augmented Generation) capabilities
- Multi-Specialist Response Generation
- Safety Guardrails and Confidence Scoring
- Text-to-Speech (TTS) for audio output
- Real-time Appointment Booking and Doctor Portal Notifications
- Consultation History Tracking
- Stunning, Fully Responsive, Animated UI/UX

================================================================================
                            TECHNOLOGY STACK
================================================================================

BACKEND:
  Framework: FastAPI 0.115.0+
  Server: Uvicorn 0.30.0+
  Database: PostgreSQL (Production) / SQLite (Local)
  ORM: SQLAlchemy 2.0.30+
  Auth: JWT (python-jose) with bcrypt hashing
  
AI/ML SERVICES:
  Speech Recognition:
    - Primary: Groq Whisper large-v3 API
    - Fallback: Google Speech Recognition (free, offline capability)
  Vision Analysis: Llama 4 Scout 17B via Groq
  LLM Reasoning: Llama 3.3 70B via Groq
  Text Embeddings: sentence-transformers (all-MiniLM-L6-v2)
  Vector DB: ChromaDB 0.5.0+ (RAG storage)
  
TTS (Text-to-Speech):
  - Primary: ElevenLabs (Premium)
  - Fallback: gTTS (free Google TTS)
  
FRONTEND:
  Framework: React (via Vite)
  Styling: Pure CSS with Glassmorphism, Advanced Animations & Mobile-First Media Queries
  Build Tool: Vite
  Routing: React Router DOM with Protected Role-Based Routes

DEPENDENCIES:
  - fastapi >= 0.115.0
  - uvicorn[standard] >= 0.30.0
  - pydantic[email] >= 2.7.0
  - sqlalchemy >= 2.0.30
  - psycopg2-binary >= 2.9.0 (PostgreSQL support)
  - chromadb >= 0.5.0
  - sentence-transformers >= 3.0.0
  - groq >= 0.15.0
  - python-jose[cryptography] >= 3.3.0
  - passlib[bcrypt] >= 1.7.4
  - gtts >= 2.5.4
  - speechrecognition >= 3.13.0
  - pydub >= 0.25.1
  - elevenlabs (optional)
  - requests >= 2.32.3
  - python-dotenv >= 1.0.0

================================================================================
                           API ENDPOINTS
================================================================================

AUTHENTICATION ROUTES (/auth):
  POST /auth/register
    - Multi-role support. Doctors must use "@doctor.com" email.

  POST /auth/login
    - Output: access_token, token_type, user info (including role)

DOCTOR ROUTES (/doctors):
  GET /doctors/
    - Fetch all registered doctors (used for booking)
  GET /doctors/me
    - Fetch current doctor's profile
  PUT /doctors/profile
    - Update doctor's professional profile (avatar, fee, specialty)

APPOINTMENT ROUTES (/appointments):
  POST /appointments/
    - Patient books an appointment with a doctor
  GET /appointments/me
    - Returns appointments for user (acts dynamically based on role)

CONSULTATION & PATIENT ROUTES (/consult, /history, /patients):
  - Standard patient endpoints for interacting with the AI engine.

================================================================================
                          HOW TO RUN THE PROJECT
================================================================================

1. Clone repository and setup venv.
2. Install dependencies: `pip install -r requirements_no_eleven.txt`
3. Set environment variables in `.env` (GROQ_API_KEY, DATABASE_URL, etc.)
4. Database will auto-migrate via SQLAlchemy on startup.
5. Run the application:
   - Backend: `uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload`
   - Frontend: `cd frontend && npm run dev`
6. Access UI at http://localhost:5173 (or Vite default port).

TROUBLESHOOTING:

Issue: "ModuleNotFoundError: No module named 'chromadb'"
Solution: pip install -r requirements_no_eleven.txt

Issue: "STT failed: Organization has been restricted"
Solution: Your Groq API key is invalid/expired. Get a new one from https://console.groq.com/keys

Issue: Frontend can't connect to backend
Solution: Ensure both services are running and check BACKEND_URL in .env

Issue: Database locked (SQLite)
Solution: Delete ai_doctor.db and restart backend (will recreate)

Issue: Port already in use
Solution: Change port in run.py or kill existing process:
  - Windows: netstat -ano | findstr :8000 вҶ’ taskkill /PID <pid>
  - Linux/Mac: lsof -i :8000 вҶ’ kill -9 <pid>
