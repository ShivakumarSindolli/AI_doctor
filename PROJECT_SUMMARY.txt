================================================================================
                              PROJECT OVERVIEW
================================================================================

AI Doctor Pro is a comprehensive full-stack application that combines:
- Speech-to-Text (STT) for capturing patient symptoms via voice
- Medical Vision Analysis for analyzing patient-uploaded images
- Intelligent Triage System to route cases to appropriate specialists
- Diagnosis Engine with RAG (Retrieval-Augmented Generation) capabilities
- Multi-Specialist Response Generation
- Safety Guardrails and Confidence Scoring
- Text-to-Speech (TTS) for audio output
- Patient Authentication & Management
- Consultation History Tracking
- Multi-turn Conversation Memory

================================================================================
                            TECHNOLOGY STACK
================================================================================

BACKEND:
  Framework: FastAPI 0.115.0+
  Server: Uvicorn 0.30.0+
  Database: SQLite (default) or PostgreSQL
  ORM: SQLAlchemy 2.0.30+
  Auth: JWT (python-jose) with bcrypt hashing
  
AI/ML SERVICES:
  Speech Recognition:
    - Primary: Groq Whisper large-v3 API
    - Fallback: Google Speech Recognition (free, offline capability)
  Vision Analysis: Llama 4 Scout 17B via Groq
  LLM Reasoning: Llama 3.3 70B via Groq
  Text Embeddings: sentence-transformers (all-MiniLM-L6-v2)
  Vector DB: ChromaDB 0.5.0+ (RAG storage)
  
TTS (Text-to-Speech):
  - Primary: ElevenLabs (Premium)
  - Fallback: gTTS (free Google TTS)
  
FRONTEND:
  Framework: React (via Vite)
  UI Library: Gradio 5.12.0+
  Build Tool: Vite
  Components: Custom React components + Gradio interface

DEPENDENCIES:
  - fastapi >= 0.115.0
  - uvicorn[standard] >= 0.30.0
  - pydantic[email] >= 2.7.0
  - sqlalchemy >= 2.0.30
  - psycopg2-binary >= 2.9.0 (PostgreSQL support)
  - chromadb >= 0.5.0
  - sentence-transformers >= 3.0.0
  - groq >= 0.15.0
  - python-jose[cryptography] >= 3.3.0
  - passlib[bcrypt] >= 1.7.4

  ├─ specialist (String) - Routed specialty

  ├─ icd_codes (Text) - Comma-separated ICD-10 codes
  ├─ flagged (Boolean) - Safety flag
  ├─ created_at (DateTime)
  └─ Relationship: user (M:1)

CONVERSATION_TURNS TABLE:
  ├─ id (PK, Integer)
  ├─ session_id (String, Indexed, FK)
  ├─ role (String) - "user" or "assistant"
  ├─ content (Text) - Message content
  ├─ created_at (DateTime)
  └─ Relationship: session (M:1)

================================================================================
                          API ENDPOINTS
================================================================================

AUTHENTICATION ROUTES (/auth):
  POST /auth/register
    - Create new user account
    - Input: email, password, full_name
    - Output: user_id, token, token_type

  POST /auth/login
    - User authentication via email/password
    - Output: access_token, token_type, user info

  POST /auth/refresh
    - Refresh expired JWT token
    - Input: refresh_token
    - Output: new access_token

PATIENT ROUTES (/patients):
  GET /patients/me
    - Get current logged-in patient profile
    - Output: Full patient details (name, age, medical history, etc.)

  PUT /patients/me
    - Update patient profile
    - Input: Medical info (age, gender, blood_type, allergies, medications)
    - Output: Updated patient object

  GET /patients/{user_id}
    - Get specific patient info (requires auth)
    - Output: Patient details

CONSULTATION ROUTES (/consult):
  POST /consult/
    - Main consultation endpoint (full pipeline)
    - Input: audio (audio file), image (optional), session_id (optional)
    - Process: STT → Triage → RAG → Diagnosis → Safety → TTS
    - Output: Complete consultation result with diagnosis, response, audio

HISTORY ROUTES (/history):
  GET /history/?limit=20
    - Retrieve past consultations for current user
    - Query: limit (default 20), offset (pagination)
    - Output: List of consultation records with summaries

  GET /history/{session_id}
    - Get specific consultation details
    - Output: Full consultation with conversation history

  GET /history/{session_id}/conversation
    - Get multi-turn conversation for a session
    - Output: Array of conversation turns (user/assistant messages)

================================================================================
                          ERROR HANDLING & FALLBACKS
================================================================================

SPEECH-TO-TEXT:
  Primary: Groq Whisper large-v3
  └─ Fallback: Google Speech Recognition (free)
     └─ Fallback: Return user-friendly error message

VISION ANALYSIS:
  Primary: Llama 4 Scout via Groq
  └─ Fallback: Continue consultation without vision analysis

TEXT-TO-SPEECH:
  Primary: ElevenLabs (premium voices)
  └─ Fallback: Google gTTS (free)
     └─ Fallback: Return response as text only

RAG RETRIEVAL:
  Primary: ChromaDB with sentence-transformers embeddings
  └─ Fallback: Continue without knowledge base context

TRIAGE/DIAGNOSIS/SPECIALISTS:
  Primary: Groq LLM APIs
  └─ Fallback: Return safe default responses with error flags

DATABASE:
  Primary: Save to database
  └─ Fallback: Rollback transaction, log error, return error message

ALL ERRORS:
  ├─ Detailed logging for debugging
  ├─ User-friendly error messages (no API errors exposed)
  ├─ Graceful degradation (continue with alternatives)
  └─ Safety flags set when errors occur

================================================================================
                          HOW TO RUN THE PROJECT
================================================================================

PREREQUISITES:
  ✓ Python 3.8+ (tested on 3.14.3)
  ✓ Node.js 16+ (for frontend)
  ✓ Virtual environment (venv or conda)
  ✓ Git

INSTALLATION STEPS:

1. Clone the repository:
   git clone <repo_url>
   cd ai_doctor_pro

2. Create & activate virtual environment:
   python -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate

3. Install dependencies:
   pip install -r requirements_no_eleven.txt
   (or with ElevenLabs: pip install -r requirements.txt)

4. Set up environment variables:
   cp .env.example .env
   # Edit .env with your API keys:
   # - GROQ_API_KEY (required)
   # - ELEVEN_API_KEY (optional)
   # - SECRET_KEY (generate: openssl rand -hex 32)
   # - DATABASE_URL (optional, defaults to SQLite)

5. Initialize database:
   # Automatically done on first backend startup
   # Or manually: python -c "from backend.database import init_db; init_db()"

6. Start both services:
   python run.py
   # Starts FastAPI backend on http://127.0.0.1:8000
   # Starts React frontend on http://127.0.0.1:7860
   # Both services run in the background

   OR run separately:
   # Terminal 1 - Backend:
   uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload
   
   # Terminal 2 - Frontend:
   cd frontend && npm run dev -- --host 127.0.0.1 --port 7860

7. Access the application:
   - Frontend UI: http://127.0.0.1:7860
   - Backend API: http://127.0.0.1:8000
   - API Docs: http://127.0.0.1:8000/docs (Swagger UI)
   - ReDoc: http://127.0.0.1:8000/redoc

TROUBLESHOOTING:

Issue: "ModuleNotFoundError: No module named 'chromadb'"
Solution: pip install -r requirements_no_eleven.txt

Issue: "STT failed: Organization has been restricted"
Solution: Your Groq API key is invalid/expired. Get a new one from https://console.groq.com/keys

Issue: Frontend can't connect to backend
Solution: Ensure both services are running and check BACKEND_URL in .env

Issue: Database locked (SQLite)
Solution: Delete ai_doctor.db and restart backend (will recreate)

Issue: Port already in use
Solution: Change port in run.py or kill existing process:
  - Windows: netstat -ano | findstr :8000 → taskkill /PID <pid>
  - Linux/Mac: lsof -i :8000 → kill -9 <pid>
